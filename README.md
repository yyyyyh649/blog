## 🚀 添加新文章（推荐方式）

只需在 `markdown/` 目录中创建一个 `.md` 文件，推送到 GitHub 后会**自动构建和发布**！

### Markdown 文件格式

每个 Markdown 文件需要在开头包含 **frontmatter**（元数据），格式如下：

```markdown
---
title: "文章标题"
date: 2024-12-20
category: "人生感悟"
excerpt: "这是文章的摘要，会显示在文章列表中..."
pinned: false
---

这里是文章正文内容...

## 小标题

更多内容...
```

### Frontmatter 字段说明

| 字段 | 必填 | 说明 | 示例 |
|------|------|------|------|
| `title` | ✅ 是 | 文章标题 | `"关于成长的思考"` |
| `date` | ✅ 是 | 发布日期，格式为 YYYY-MM-DD | `2024-12-20` |
| `category` | ❌ 否 | 文章分类 | `"人生感悟"` / `"年度计划"` / `"随想杂记"` |
| `excerpt` | ❌ 否 | 文章摘要，不填则自动截取正文前100字 | `"这是摘要..."` |
| `pinned` | ❌ 否 | 是否置顶，默认为 false | `true` 或 `false` |

### 📌 如何设置文章置顶

在 frontmatter 中将 `pinned` 设置为 `true` 即可：

```markdown
---
title: "这是一篇置顶文章"
date: 2024-12-20
category: "年度计划"
excerpt: "这篇文章会显示在列表最前面"
pinned: true
---
```

**置顶规则：**
- 置顶文章会显示在文章列表的最前面
- 多篇置顶文章之间按日期降序排列
- 置顶文章会有特殊的视觉标识（📌 图标和边框高亮）

### Markdown 语法支持

支持常见的 Markdown 语法：

- **标题**：`# 一级标题` `## 二级标题` 等
- **粗体**：`**粗体文字**`
- **斜体**：`*斜体文字*`
- **引用**：`> 引用文字`
- **链接**：`[链接文字](URL)`
- **图片**：`![图片描述](图片URL)`
- **列表**：`- 无序列表` 或 `1. 有序列表`
- **行内代码**：`` `代码` ``

### 文件命名规范

- 文件名将作为文章的 URL slug
- 建议使用英文、数字和连字符
- 例如：`my-first-post.md` → 生成 `posts/my-first-post.html`

date: 2024-12-25

写博客是一种很好的记录和思考方式...

## 未来计划

- 记录生活中的点滴

## 自动构建说明

当你向 `markdown/` 目录推送新的 `.md` 文件时：

1. GitHub Actions 会自动运行构建脚本
2. 脚本将 Markdown 转换为 HTML 页面
3. 自动更新首页和文章列表页
4. 文章按日期排序（置顶文章优先）
5. 更改会自动提交并部署

## 本地构建

如果需要本地预览，可以运行：

```bash
node scripts/build.js
```

然后用浏览器打开 `index.html` 查看效果。

## 部署

### GitHub Pages

1. 进入仓库设置 (Settings)
2. 找到 Pages 选项
3. 在 Source 中选择 `main` 分支
4. 保存后等待部署完成

### 本地预览

直接用浏览器打开 `index.html` 文件即可预览。

## 自定义

- 修改 `css/style.css` 中的 CSS 变量可以调整颜色主题
- 修改 `about.html` 添加个人信息
- 替换文章内容为自己的文字
